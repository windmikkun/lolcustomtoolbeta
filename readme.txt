LoLカスタムチーム分けツール(β)
    •    参加者の 名前 / ランク（表記） / メインレーン / サブレーン を登録して、チームを自動編成します。
    •    目的は次の2つを同時に満たすことです
    
チーム総合戦力ができるだけ均等
各レーンの対面差（TOP/JG/MID/ADC/SUP）ができるだけ小さくなるように

ランクの扱い
    •    入力は ランク表記（例：Gold II）
    •    内部ではスコアに変換して計算します

レーン適性（重要）
    •    メイン or サブで出る場合：ペナルティなし（同格扱い）
    •    それ以外のレーン：tier-2扱い（-2 division相当）
    •    例：Gold I がオフレーンの場合 → Gold III 相当として計算

MATCHUP_WEIGHT（対面重視度）
    •    スライダーで 対面差の重視度を調整できます（0〜4）
    •    目安
    •    0：総合戦力優先（対面は荒れやすい）
    •    1〜2（おすすめ）：総合と対面のバランス型
    •    3：対面重視（レーン差をかなり抑える）
    •    4：対面最優先（総合差が多少出る場合あり）

    •    2チーム（10人） / 3チーム（15人） / 4チーム（20人） に対応
    •    参加者が必要人数より多い場合は 自動抽出します
    •    例：12人いるが2チームにしたい → 10人を自動選出して編成

メイン/サブのみ割り当てモード
    •    チェックボックスONで、各プレイヤーはメインまたはサブにのみ配置
    •    配置できない場合は結果下部に「割り当てできなかった参加者」として表示
    •    オフレーン強制なし（空席が残ることがあります）

未選出/未割当の表示
    •    人数超過で落ちた人を「抽出されなかった参加者」として表示
    •    strictモードで置けなかった人を「割り当てできなかった参加者」として表示

バルク入力
    •    複数行のまとめ入力に対応（CSV風）
    •    フォーマットエラー時は行番号付きでアラート表示

データ保存
    •    参加者リストはブラウザのlocalStorageに自動保存・復元
    •    ページ更新後も入力済みメンバーが維持されます

注意事項
    •    完全に公平を保証するものではありません（メタ、得意チャンプ、相性、デュオ経験などは考慮しません）
    •    Master+ は簡易扱い（LPは未対応）なので、必要なら運営が微調整します
    •    「対面差を抑える」と「総合差を抑える」はトレードオフです
→ 迷ったら MATCHUP_WEIGHT = 2 or 3 推奨
